... oder "modulate Arithmetik" ist das *Rechnen mit Resten bei der division ganzer Zahlen*.
Vertraut ist das vom Rechnen mit Uhrzeiten her: 6 Stunden nach 21 Uhr ist nicht 27 Uhr, sondern wieder 3 Uhr – dabei ist 3 der Rest von 27 bei Division durch 24. Es wurd **modulo** 24 gerechnet. Entsprechend wird bei den Minuten und Sekunden modulo 60 gerechnet.
Es zeigt sich, dass die modulare Arithmetik noch für viele andere Dinge von Nutzen ist, sowohl in der Mathematik, als auch in anderen Anwengungen.

Wir fixieren eine Zahl $n \in \mathbb{N}$, den **Modulus**. Zwei ganze Zahlen $a, b \in \mathbb{Z}$ heißen **kongruent modulo $n$**,  wenn sie bei Division durch $n$ denselben Rest ergeben. Wir schreiben in diesem Fall $$a \equiv b (\text{mod } n).$$
>[!example] 12.22 Beispiel
>Es gelten z.B. $$9 \equiv 5 (\text{mod } 4), \ 14 \equiv 0 (\text{mod } 7), \ 6 \equiv 72 \equiv 39 (\text{mod } 11).$$ 

>[!TIP] 12.23 Lemma
>Für $a, b \in \mathbb{Z}$ und $n \in \mathbb{N}$ gilt $$a \equiv b (\text{mod } n) \Leftrightarrow n|(a-b).$$
>In Worten:
>Genau dann sind zwei Zahlen **kongruent modulo $n$**, wenn ihre *Differenz durch $n$ teilbar* ist.
>Insbesondere gilt $a \equiv 0 (\text{mod } n)$, genau dann, wenn $a$ durch $n$ teilbar ist.

>[!TIP] **12.24 Satz** ***Eigenschaften der Kongruenz***
>Seien $k, n \in \mathbb{N}$ und $a, a', b, b', c \in \mathbb{Z}$.
>1. $a\equiv a (\text{mod } n)$   *Reflexivität*
>2. $a \equiv b (\text{mod } n) \Leftrightarrow b \equiv a (\text{mod } n)$   *Symmetrie*
>3. $(a \equiv b (\text{mod } n) \land b \equiv c (\text{mod } n)) \implies a \equiv c (\text{mod } n)$   *Transitivitöt*
>4. $(a \equiv a' (\text{mod } n) \land b \equiv b' (\text{mod } n)) \implies a+b \equiv a'+b' (\text{mod } n)$
>5. $(a \equiv a' (\text{mod } n) \land b \equiv b' (\text{mod } n)) \implies ab \equiv a'b' (\text{mod } n)$
>6. $(c,n$ teilerfremd $\land ca \equiv cb (\text{mod }n)) \implies a \equiv b (\text{mod }n)$ 
>7. $a \equiv b (\text{mod }kn) \implies a \equiv b (\text{mod }n)$

>[!def] Definition
>Sei $a \in \mathbb{Z}$. Wir schreiben $$[a]_{n} = \{ b \in \mathbb{Z} \ | \ a \equiv b (\text{mod } n) \}.$$
>für die Menge aller Zahlen, die zu $a$ kongruent sind, die **Kongruenzklasse von $a$ modulo $n$.**
>
>Wenn zwei Zahlen kongruent sind, dann sind die Kongruenzklassen *gleich*: $$a \equiv b (\text{mod } n) \Leftrightarrow [a]_{n} = [b]_{n}.$$

>[!TIP] 12.26 Lemma 
>Es sei $n \in \mathbb{N}$. Die *Kongurenzklassen* von modulo n sind $$[0]_{n}, [1]_{n}, [2]_{n}, \dots, [n-1]_{n}$$ und $\mathbb{Z}$ ist die **disjunkte** Vereinigung dieser Kongurenzklassen.
>___
>*BEWEIS*
>![[Pasted image 20251219113017.png]]

Wir schreiben $$\mathbb{Z}/n = \{ [0]_{n}, [1]_{n}, \dots, [n-1]_{n} \}$$für die Menge der verschiedenen Kongurenzklassen modulo $n$.

---

>[!TIP] **12.27** Satz 
>Die Menge $\mathbb{Z} \text{/} n$ wird mit der *Addition* bzw. *Multiplikation* $$[a]_{n} + [b]_{n} = [a+b]_{n} \text{  bzw.  } [a]_{n} \cdot [b]_{n} = [ab]_{n}$$ zu einem **kommitetiven Ring**. Die Null $[0]_{n}$ und die Eins $[1]_{n}$.
>___
>*BEWEIS*
>![[Pasted image 20251224141321.png]]
>![[Pasted image 20251224141343.png]]

Für jede natürliche Zahl $n$ haben wir damit einen Ring mit $n$ Elementen konstruiert, dessen Elemente die Kongurennzklassen $[0]_{n}, [1]_{n}, \dots, [n-1]_{n}$ sind. 
Wenn klar ist, dass wir in $\mathbb{Z}/n$ arbeiten, assen wir die Klammern und den Index $n$ weg.
